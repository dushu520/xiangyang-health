# 向阳健康 - EdgeOne Pages 部署指南

## 一、准备工作

### 需要的信息
在开始部署前，请准备好以下信息：

| 项目 | 说明 | 示例 |
|------|------|------|
| 后端服务器地址 | 您的云服务器 IP 或域名 | `https://api.xiangyang.com` 或 `http://1.2.3.4:3000` |
| GitHub 账号 | 需要有项目仓库的访问权限 | - |

---

## 二、登录 EdgeOne Pages

### 步骤 1：访问控制台
打开浏览器，访问：
```
https://pages.edgeone.ai
```

### 步骤 2：登录账号
使用腾讯云账号登录（如果没有，需先注册腾讯云账号）

---

## 三、创建项目

### 步骤 3：进入创建页面
点击页面上的 **「创建项目」** 按钮

### 步骤 4：选择导入方式
选择 **「从 Git 仓库导入」**

### 步骤 5：授权 GitHub
1. 点击 **「授权访问」** GitHub
2. 在弹出的 GitHub 页面中确认授权
3. 授权完成后会自动返回 EdgeOne Pages

### 步骤 6：选择仓库
在仓库列表中找到并选择：
```
dushu520/xiangyang-health
```

### 步骤 7：选择分支
在分支下拉菜单中选择：
```
feature/frontend-backend-separation
```
> 注：如果项目已合并到主分支，选择 `master` 或 `main`

---

## 四、配置构建设置

### 步骤 8：填写构建信息
在构建设置页面，按以下表格填写：

| 配置项 | 填写内容 | 说明 |
|--------|---------|------|
| **构建命令** | `pnpm run build` | 编译前端代码的命令 |
| **输出目录** | `dist/public` | 构建完成后要部署的文件夹 |
| **Node 版本** | `20.18.0` | 建议选 20.x 版本 |

### 步骤 9：选择包管理器
在 **包管理器** 选项中，选择：
```
pnpm
```

---

## 五、配置环境变量

### 步骤 10：添加环境变量
点击 **「环境变量」** 区域的 **「添加」** 按钮

填写以下信息：

| 变量名 | 变量值 |
|--------|--------|
| `VITE_API_BASE_URL` | 您的后端服务器地址 |

**后端服务器地址示例：**
- 如果有域名：`https://api.xiangyang.com`
- 如果只有 IP：`http://123.45.67.89:3000`

> ⚠️ **重要**：请确保地址正确，前端需要通过这个地址访问后端 API

### 步骤 11：保存配置
确认所有信息填写无误后，点击 **「保存」** 或 **「开始部署」**

---

## 六、等待部署完成

### 步骤 12：查看构建进度
1. 系统会自动开始构建，页面会显示构建日志
2. 等待构建完成（通常需要 2-5 分钟）
3. 构建成功后，会显示绿色的 **「部署成功」** 提示

### 步骤 13：获取访问地址
部署成功后，系统会分配一个默认域名：
```
https://您的项目名称.pages.edgeone.app
```

点击该链接即可访问网站。

---

## 七、验证网站

### 检查清单
打开网站后，请检查以下功能是否正常：

- [ ] 首页能正常打开，显示「向阳健康」
- [ ] 文章列表正常显示（健康前沿、健康讲堂、健康科普）
- [ ] 图片能正常加载
- [ ] 点击文章可以进入详情页
- [ ] 后台登录功能正常（访问 `/admin/login`）

### 常见问题

#### 问题 1：页面打开空白
**原因**：后端 API 地址配置错误

**解决**：
1. 进入项目 → 设置 → 环境变量
2. 检查 `VITE_API_BASE_URL` 是否正确
3. 修改后重新部署

#### 问题 2：图片不显示
**原因**：后端服务器未配置跨域或图片路径错误

**解决**：
- 联系后端开发人员检查服务器配置
- 参考后端部署文档中的 Nginx 配置

#### 问题 3：构建失败
**原因**：可能是依赖安装失败

**解决**：
1. 点击 **「重新部署」**
2. 如果多次失败，检查构建命令是否正确（应为 `pnpm run build`）

## 八、配置自定义域名（可选）

### 步骤 14：添加域名（如需要）

如果您有自己的域名，可以绑定到该项目：

1. 进入项目 → **域名管理**
2. 点击 **「添加域名」**
3. 输入您的域名，如：`www.xiangyang.com`
4. 按提示在 DNS 服务商添加 CNAME 记录
5. 等待 SSL 证书自动签发（约 5-10 分钟）

---

## 九、后续更新

### 自动部署
项目已配置自动部署，当代码推送到 GitHub 后会自动重新构建：

- 推送到 `feature/frontend-backend-separation` 分支 → 更新预览环境
- 推送到 `master` 分支 → 更新生产环境

### 手动重新部署
如需手动更新：
1. 进入 EdgeOne Pages 控制台
2. 选择项目
3. 点击 **「重新部署」** 按钮
